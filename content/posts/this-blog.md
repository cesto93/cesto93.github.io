Welcome to my blog!

I don't know who would read this but I am a casual developer in the AI world.
Well about my background, I have a degree in IT.
I have started as a C developer, to then be a Go developer, to be primarly a solution architect.
When i was in university i would not even know "solution architect" was a job title.
And i think this is somehow the path of the IT fields, from old low level programming, to a new programming language, then
to a high level cloud job.

And now i am somehow writing a blog using a project i find asking gemini, then installing it using opencode.
Probably i would even use opencode to check my english (I am not a native english speaker, probably even opencode is not).

So i think the IT fields has changed, the bigger change i have ever seen.
Two years ago I was thinking coding was the hard part, now coding can be done by almost anyone.
Coding is not all in the Software Development but was a big part.

I think human has always tryed a way to let maching write code, you could say a C compiler write assembly code.
But I think the best example would be the Java setter and getter generator in the IDE or the autocomplete in the IDE, without
that 90% of the Java programmers would cry.
So then followed copilot that would autocomplete not by a word but multiple words a time.
And then came the new copilot that would write code from a prompt in the IDE (no chatgpt copy paste).
And finally came claude code, opencode and all the cli.

Why i have differentied the cli from the initial copilot? Because they would also install dependency, launch tests, and launch commands.

What would be the next step?
Maybe i start to see it, i have still not tryed it.
A fully virtualized system where you write user stories, you do not check the code, and get a notification when the story is completed, it run all in the background.
This way you can slash a stream of task to the agent and the agent does a bunch of stuff.
Then you would need to check the results.
You would be able to follow more projects surely, but I think the multi-tasking takes a toll.
You can't focus on somenthing because you alway run around from a task to another.

We'll see what sticks!
