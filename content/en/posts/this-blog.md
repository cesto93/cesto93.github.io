Welcome to my blog!

I don't know who would read this but I am a casual developer in the AI world.
As for my background, I have a degree in IT.
I started as a C developer, then became a Go developer, and now I'm primarily a solution architect.
When I was in university, I didn't even know "solution architect" was a job title.
And I think this mirrors the path of IT in general — from old low-level programming, to a newer language, then
to a high-level cloud job.

And now I'm somehow writing a blog using a project I found by asking Gemini, then installing it with opencode.
I'll probably even use opencode to check my English (I'm not a native English speaker — probably opencode isn't either).

So I think the IT field has changed more than anything I've ever seen.
Two years ago I thought coding was the hard part; now coding can be done by almost anyone.
Coding isn't everything in software development, but it was a big part.

I think humans have always tried to make machines write code — you could say a C compiler writes assembly code.
But I think the best examples are the Java getter/setter generator in the IDE and autocomplete — without
those, 90% of Java programmers would cry.
Then came Copilot, which autocompletes not one word but multiple words at a time.
And then came the new Copilot that writes code from a prompt inside the IDE (no more ChatGPT copy-paste).
And finally came Claude Code, opencode, and all the CLI tools.

Why did I separate the CLI tools from the earlier Copilot? Because they also install dependencies, run tests, and execute commands.

What's next?
Maybe I'm starting to see it, though I haven't tried it yet.
A fully virtualized system where you write user stories, don't look at the code, and get a notification when the story is done — it all runs in the background.
This way you can throw a stream of tasks at the agent and it does a bunch of stuff.
Then you'd need to check the results.
You could follow more projects, sure, but I think multitasking takes a toll.
You can't focus on something because you're always running from one task to another.

We'll see what sticks!
